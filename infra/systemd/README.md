# systemd and host helpers

This directory contains host-level operational scripts for diagnostics, tuning, backup and rollback.

No unit or timer files are currently defined here. If they are added later, place them under a `units/` or `timers/` subdirectory and keep service-specific application logic outside `infra/`.

Ollama host configuration is generated by the central config resolver instead
of being edited manually:

```bash
make ollama-host-config
make ollama-host-apply
```

The generated files live in `.local/generated/ollama-host/`. The apply command
installs only the ai-local-owned `90-ai-local.conf` systemd drop-in and restarts
the native `ollama` service.
